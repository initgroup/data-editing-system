PL/SQL 바인드 변수 규칙

Parameter List의 Parameter 명칭과 시스템 예약 변수를 PL/SQL 스크립트에서 바인드 변수 또는 문자열 치환 키로 사용할 수 있습니다.

  • 바인드 변수명 규칙: Parameter 명칭을 camelCase로 변환해 사용합니다. 예: ABC_DEF -> :abcDef, P_MODEL_NAME -> :pModelName.
  • 작성 방법: PL/SQL에서는 바인드 변수 앞에 콜론을 붙입니다. 예: P_MODEL_NAME => :pModelName.
  • 동적 문자열 치환: /* --DYNAMIC_MODEL_NAME-- */처럼 작성하면 /* --와 -- */ 사이의 이름이 Parameter 명칭과 완전히 같을 때 해당 값이 문자열 그대로 치환됩니다.
  • 현재 Job 예약 변수: :INIT$TargetOwner, :INIT$TargetTable, :INIT$ResultOwner, :INIT$ResultTable은 실행 시 현재 Job/Node의 Target/Result owner/table 값으로 자동 전달됩니다.
  • Flow 선행 Node 예약 변수: M04001 Flow 실행에서는 :INIT$PreTargetOwner, :INIT$PreTargetTable, :INIT$PreResultOwner, :INIT$PreResultTable을 사용할 수 있습니다. 연결된 선행 Node의 Target/Result owner/table 값이 자동 전달됩니다.
  • Generate PL/SQL: Parameter 기본값이 있어도 스크립트에는 위 규칙의 바인드 변수로 생성합니다. 기본값과 예약 변수 값은 실행 직전 Runtime Bind 값으로 확인하고 수정할 수 있습니다.
  • Run now / Queue Batch: 실행 시 Parameter List 기본값과 Runtime Bind 값을 처리하고, 시스템 예약 변수는 실제 Job/Node 정보로 자동 덮어쓴 뒤 실행합니다.
  • 예시: P_OWNER => :INIT$PreResultOwner, P_TABLE => :INIT$PreResultTable, P_RESULT_OWNER => :INIT$ResultOwner, P_RESULT_TABLE => :INIT$ResultTable
