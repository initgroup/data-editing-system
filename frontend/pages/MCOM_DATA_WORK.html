PL/SQL 바인드 변수 규칙

Generate PL/SQL은 Parameter List의 Default 값을 먼저 확인한 뒤, 필요한 바인드 변수를 자동으로 만듭니다.

1. Default에 :가 있으면 그대로 사용 :INIT$TargetOwner -> P_TARGET_OWNER => :INIT$TargetOwner
2. Default에 :가 없으면 Parameter명을 camelCase로 변환 P_DYNAMIC_MODEL_NAME -> :pDynamicModelName
3. 실행 시 실제 값으로 바인딩 Run saved work / Queue saved work는 저장된 Work를 실행하고, Test current draft는 현재 화면값을 저장하지 않고 1회 실행합니다.

자동 예약 변수

:INIT$TargetOwner현재 Target Owner :INIT$TargetTable현재 Target Table :INIT$ResultOwner현재 Result Owner :INIT$ResultTable현재 Result Table :INIT$RunSourceTypeDATA_WORK/FLOW_WORK :INIT$RunId현재 실행 이력 ID

생성 예시

P_TARGET_OWNER       => :INIT$TargetOwner
P_TARGET_TABLE       => :INIT$TargetTable
P_RUN_SOURCE_TYPE    => :INIT$RunSourceType
P_RUN_ID             => :INIT$RunId
P_DYNAMIC_MODEL_NAME => :pDynamicModelName
P_PREDICTION_METHOD  => :pPredictionMethod
