Input and Output Contracts

Each node exposes named inputs and outputs. A port can carry a table, scalar value, array, JSON object, or model result.

  • TABLE: Usually an owner/table name, temporary table, or result table produced by a prior node.
  • VALUE: A scalar runtime value such as threshold, owner, date, or option code.
  • ARRAY: A list such as selected columns, discovered rules, or validation messages.
  • SOLID EDGE: Runs the downstream node only when the connected upstream node finishes with SUCCESS.
  • DASHED EDGE: Runs the downstream node after the connected upstream node finishes, regardless of SUCCESS or FAILED status.
  • JOIN: A node can wait for multiple upstream edges; each solid edge must be SUCCESS, while dashed edges only need to be finished.
